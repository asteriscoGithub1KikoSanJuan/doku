git - Der einfache Einstieg

eine einfache Anleitung, um git zu lernen. Kein Schnick-Schnack

Installation unter Linux

Wenn Sie die grundlegenden Git-Tools auf Linux über ein Binär-Installationsprogramm installieren möchten, können Sie dies in der Regel über das grundlegende Paketverwaltungs-Tool mit Ihrer Distribution durchführen. Wenn Sie auf einer Debian-basierten Distribution wie Ubuntu sind, versuchen Sie apt-get:

$ sudo apt-get install git-all

Für weitere Optionen gibt es Anweisungen für die Installation auf verschiedenen Unix-Flavours auf der Git-Website unter

Neues Repository erstellen

erstelle ein neues Verzeichnis, öffne es und führe

git init

aus, um ein neues git-Repository anzulegen.

Ein Repository auschecken

erstelle eine Arbeitskopie, indem du folgenden Befehl ausführst:

git clone /pfad/zum/repository

Falls du ein entferntes Repository verwendest, benutze:

git clone benutzername@host:/pfad/zum/repository

ich arbeite mit folgenden Projekten:

ADP: kiko@Satelite:/opt/lampp/htdocs/ADP$ git clone https://github.com/XaviAngular/ADP.git

materialize: kiko@Satelite:/opt/lampp/htdocs/materialize$ git clone https://github.com/XaviAngular/materialize.git

work: kiko@Satelite:/opt/lampp/htdocs/work$ git clone https://github.com/asteriscoGithub1KikoSanJuan/materialize.git

Workflow

Dein lokales Repository besteht aus drei "Instanzen", die von git verwaltet werden. Die erste ist deine Arbeitskopie (devel), welche die echten Dateien enthält. Die zweite ist der Index (master), welcher als Zwischenstufe agiert und zu guter Letzt noch der HEAD, der auf deinen letzten Commit zeigt.
